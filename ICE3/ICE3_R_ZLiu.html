Linear regression

linear regression models are a good starting point for all kinds of regression tasks. Such models are popular because they can be fit very quickly (blazing fast!), and are very interpretable. You are probably familiar with the simplest form of a linear regression model (i.e., fitting a straight line to data) but such models can be extended to model more complicated data behavior.

If you need a memory refresher about what linear regression is, here is a good intro video to watch: https://youtu.be/nk2CQITm_eo

Simple linear regression

We will start with the most familiar linear regression, a straight-line fit to data.

\[y = a + bx\]

where \(a\) is commonly known as the slope, and \(b\) is commonly known as the intercept.

Once we have the data ready, then fitting the model is just easy-peasy by calling the lm() function.

In lm() function, you will first need to state the proposed model and use ~ to link the predicted variable and predictor variable together. Then, tell R that the data is from graduationCollege. Alternatively, you can use $ to select the variables as what we have done in ICE2, but it is not necessary since we have already specified the data = parameter. Don’t forget to assign the model to one object. In this case, we use model to store the model information. Feel free to call it by different names. In practice, I prefer explicit names such as collegeEnrollModel. It will really help you remember things you are doing. See more style guide in here: http://adv-r.had.co.nz/Style.html

A side note, normally the dataset you work with might not have the spaces in column names, so you don’t always have to use ``.

Once you have build the model, you can call summary() to view the model statistics. R provide pretty comprehensive R summary that contains pretty much all you need. Notice that model here is a list object. It contains a lot of information, which might be useful to you for different purposes. Feel free to use $ to see what is in it.

model <- lm(`college enroll 2010-11`~`graduation 2010-11`, data = graduationCollege)

If you just need to coefficient information, then you can use coef() to extract. In the simple linear regression case, you will only have to values. Therefore, we can use coef(model)[1] to extract the intercept and coef(model)[2] to extract the slope.

abline() cannot work alone. This allows you to draw horizontal, vertical, or sloped lines. To draw the fitted line, you can specify a as the intercept and b as the slope, and col as the color of the line. Or just simply use abline(model). But note that, abline() does not work alone. You have to run both lines together.

plot(graduationCollege)

abline(a = coef(model)[1], b = coef(model)[2], col = "red") # Or you can simply do abline(model)

## A more realistic example with multiple independent variables

Here we are going to look at some interaction data from students watching on a class video. A project was proposed to collect data from student video watching, a sample of this data is available in the file ICE3data.csv.

  • stid = student id
  • year = year student watched video
  • participation = whether or not the student opened the video
  • watch.time = how long the student watched the video for
  • confusion.points = how many times a student rewatched a section of a video
  • key.points = how many times a student skipped or increased the speed of a video

Let’s see if we can build a regressor to predict how long the student will watch a video for.

For future readings on more advanced issues in regression:

  • regularization in Python: Regularization helps to solve over fitting problem. Simple model will be a very poor generalization of data. At the same time, complex model may not perform well in test data due to over fitting. We need to choose the right model in between simple and complex model (see here: https://www.r-bloggers.com/2017/07/machine-learning-explained-regularization/)
  • polynomial and non-linear regression: Linearity is not always the case, so sometime we need non-linear models such as polynomial regression (see here:https://datascienceplus.com/fitting-polynomial-regression-r/) or other non-linear model such as decision tree (see here: https://www.datacamp.com/community/tutorials/decision-trees-R)
  • variable selection: In real world analytics, we often come across a large volume of candidate regressors, but most end up not being useful in regression modeling. Finding the most appropriate set of regressors is a variable selection issue. (see here: https://www.statology.org/stepwise-regression-r/)
