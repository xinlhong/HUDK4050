Linear regression¶

linear regression models are a good starting point for all kinds of regression tasks. Such models are popular because they can be fit very quickly (blazing fast!), and are very interpretable. You are probably familiar with the simplest form of a linear regression model (i.e., fitting a straight line to data) but such models can be extended to model more complicated data behavior.

If you need a memory refresher about what linear regression is, here is a good intro video to watch: https://youtu.be/nk2CQITm_eo

Simple linear regression¶

We will start with the most familiar linear regression, a straight-line fit to data.

$$y = a + bx$$

where $a$ is commonly known as the slope, and $b$ is commonly known as the intercept.

Here is an example from our ICE1_Data.csv:

A more realistic example with multiple independent variables¶

Here we are going to look at some interaction data from students watching on a class video. A project was proposed to collect data from student video watching, a sample of this data is available in the file ICE3data.csv.

  • stid = student id
  • year = year student watched video
  • participation = whether or not the student opened the video
  • watch.time = how long the student watched the video for
  • confusion.points = how many times a student rewatched a section of a video
  • key.points = how many times a student skipped or increased the speed of a video

For demonstration purpose, let's just use all the variables to predict watch.time.

The process is no different than the simple linear model. Just this time, you have multiple Xs:

$$video watch = b_0 + b_1 * participation + b_2 * confusion points + b_3 * key points$$

You will have to prepare a nx3 array for all the Xs and then again the to-be-predicted value -- watchTime

Here I am using a different way to create the X matrix because it contains multiple Xs.

This type of result can be hard to read sometimes. So, you can try a different Python implementation on linear regression -- OLS from statsmodels (which means ordinary least sqaure, the algorithm used to estimate each bs in linear regression) . The code are different, but the idea behind is the same. You put the variables into a model, then fit a "line/plane/hyperplane" where minimize the error and obtain the model parameters. OLS provides a slightly more comprehensive model summary for us.

For future readings on more advanced issues in regression:

  • regularization in Python: Regularization helps to solve over fitting problem. Simple model will be a very poor generalization of data. At the same time, complex model may not perform well in test data due to over fitting. We need to choose the right model in between simple and complex model (see here: https://harish-reddy.medium.com/regularization-in-python-699cfbad8622).
  • polynomial and non-linear regression: Linearity is not always the case, so sometime we need non-linear models such as polynomial regression (see here:https://towardsdatascience.com/polynomial-regression-bbe8b9d97491) or other non-linear model such as decision tree (see here: https://www.geeksforgeeks.org/python-decision-tree-regression-using-sklearn/)
  • variable selection: In real world analytics, we often come across a large volume of candidate regressors, but most end up not being useful in regression modeling. Finding the most appropriate set of regressors is a variable selection issue. (see here: https://www.datasklr.com/ols-least-squares-regression/variable-selection)
